# In-App Disclosure Copy Recommendations

These recommendations are documentation only. They do not change Android resources or app behavior.

## Device Admin Onboarding

Recommended copy:

LockWitness needs Device Admin access to receive failed unlock attempt callbacks on this device. Enable this only on a device you own or manage. You can disable Device Admin access in Android system settings.

Recommended controls:

- Primary action: Enable Device Admin
- Secondary action: Not now

## Camera Permission

Recommended copy:

Camera permission lets LockWitness capture front-camera photo or short video evidence for enabled failed unlock incidents and manual diagnostics. Video evidence is recorded without audio.

Recommended controls:

- Primary action: Grant Camera Permission
- Secondary action: Keep disabled

## Location Permission

Recommended copy:

Location permission lets LockWitness attach a location snapshot to enabled failed unlock incidents. If location is unavailable, the incident remains saved and records the location status.

Recommended controls:

- Primary action: Grant Location Permission
- Secondary action: Keep disabled

## Export And Share

Recommended copy:

Exports are created locally and started by you. A share action opens the Android chooser so you can select a destination app.

Recommended controls:

- Primary action: Export Evidence
- Secondary action: Cancel

## Ads And Pro

Recommended copy:

Free mode keeps core monitoring and photo evidence available. Pro gates optional features such as video, location, unlimited history, export, and ad removal. If billing is unavailable, the app remains usable in Free mode.

## General Copy Rules

- State what permission is used for before requesting it.
- Keep every permission explanation tied to a visible setting or user action.
- Avoid claims that imply remote monitoring, background transfer, or use on another person's device.
- Keep Settings text consistent with the Play Store listing and privacy policy.
